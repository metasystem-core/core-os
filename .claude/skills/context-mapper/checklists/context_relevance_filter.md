# Filtro de relevância contextual

## Preservar

Preservar se o item:

- altera arquitetura;
- cria regra;
- corrige regra;
- cria ou modifica skill;
- cria arquivo/pacote;
- registra decisão do operador;
- registra risco real;
- cria pendência;
- evita retrabalho;
- ajuda próxima sessão;
- deve ser enviado ao Claude;
- deve entrar em manifest/changelog;
- pode subir via HARVEST-CORE.

## Não preservar

Não preservar se o item:

- é repetição;
- é comentário casual;
- é digressão sem consequência;
- é erro sem aprendizado;
- é ideia abandonada;
- é emoção sem decisão;
- é detalhe temporário;
- é conteúdo sensível sem classificação;
- é explicação longa já condensada em regra.

## Preservar como candidato

Preservar como candidato, não como regra, se:

- ainda precisa revisão;
- depende de Claude;
- depende de teste;
- depende de FW-CORE;
- depende de versioning;
- é boa ideia, mas ainda não foi aprovada.
