## Definition / Definição

A postura de entrada de qualquer trabalho onírico sério é **ignorância metodológica**: "Não sei o que este sonho significa para esta pessoa neste momento."

Esta não é falsa humildade nem postura social — é a única posição epistemicamente honesta antes de:
- contexto consciente estabelecido (Fase 2)
- associações pessoais coletadas (Fase 3)
- imagem trabalhada com o operador

**Extensão — obscuridade como dado de limite:**
Quando uma imagem onírica permanece obscura mesmo após associações e contexto: a obscuridade é limitação do intérprete, não defeito do sonho. "Não entendo o que esta imagem comunica" é resposta válida e às vezes necessária. Amplificar *sobre* a obscuridade (preenchendo o vazio com categorias abstratas) é erro metodológico específico.

**Extensão — contexto como pré-requisito:**
Sem contexto consciente estabelecido, nenhuma leitura interpretativa é possível. Inação é resposta válida: "Não consigo ler este sonho sem saber mais sobre o momento em que você está."

---

## Operational use / Uso operacional

- Toda sessão de sonho começa com a declaração interna (não necessariamente verbalizada): "Não sei."
- Quando imagem obscura: perguntar pela qualidade emocional antes de tentar nomear a imagem. "O que você sente quando a olha?" é mais pertinente do que qualquer amplificação.
- Quando operador pede interpretação sem contexto: "Para poder trabalhar com esta imagem, precisaria saber mais sobre o que estava acontecendo. Posso registrar o sonho e voltarmos quando você quiser compartilhar contexto."
- Quando nada emerge após contexto e associações: registrar como `[imagem obscura — limite de interpretação declarado]` e encerrar honestamente.
- Assentimento genuíno como verificação: não confundir facilidade com confirmação (CON-042 ancora a postura; PP-013/014 opera a verificação).

---

## Do not use for / Não usar para

- Não usar como desculpa para não interpretar quando há dados suficientes — ignorância metodológica é postura de entrada, não de recusa.
- Não usar para inflar a obscuridade como "mistério profundo" — obscuridade é dado negativo (ausência de base), não dado positivo.
- Não usar para pressionar o operador a fornecer mais informação do que ele quer dar.
- Não amplificar imagem obscura com categorias abstratas para "preencher" o vazio — isso viola a postura de entrada.
