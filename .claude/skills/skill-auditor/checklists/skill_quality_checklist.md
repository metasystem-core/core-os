# Checklist de qualidade de skill

## Identidade

- [ ] name claro
- [ ] description útil
- [ ] status
- [ ] core_type
- [ ] priority
- [ ] domain
- [ ] risk_level
- [ ] source_policy quando necessário

## Escopo

- [ ] O que faz
- [ ] O que não faz
- [ ] Quando usar
- [ ] Quando não usar
- [ ] Limites
- [ ] Gatilhos

## Operacionalidade

- [ ] Gera artefato útil?
- [ ] Tem output contract?
- [ ] Tem modo de resposta?
- [ ] Tem templates?
- [ ] Tem checklist?
- [ ] Tem subskills quando o domínio é complexo?
- [ ] Tem bloqueios?
- [ ] Tem critérios de escalonamento?

## Segurança

- [ ] Tem watchdog?
- [ ] Declara lacunas?
- [ ] Evita overconfidence?
- [ ] Trata fonte/validade?
- [ ] Protege dados sensíveis?

## Runtime

- [ ] Alguma parte deveria virar código?
- [ ] Alguma parte deveria virar template?
- [ ] Alguma parte deveria virar teste?
- [ ] Alguma parte deveria virar schema?

## Integração

- [ ] Conversa com FW-CORE?
- [ ] Conversa com META-ROUTER?
- [ ] Conversa com CONFLICT-SCAN?
- [ ] Conversa com QA/VERSIONING?
