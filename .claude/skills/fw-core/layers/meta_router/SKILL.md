# SKILL — META-ROUTER / Reconhecimento Inicial de Demanda

## Função

Camada acima dos projetos.  
Antes de responder, reconhece o tipo de demanda, domínio, risco, estado provável do usuário e profundidade necessária.

## Rodar quando

Sempre que uma nova demanda chegar.

## Não fazer

- Não resolver tudo.
- Não fazer análise profunda.
- Não substituir FW-CORE.
- Não transformar todo pedido em decisão existencial.

## Checklist

1. Qual é o tipo de demanda?
   - decisão;
   - análise;
   - documento;
   - prompt;
   - cálculo;
   - desabafo;
   - estudo;
   - migração;
   - revisão;
   - dúvida técnica;
   - dúvida jurídica;
   - investimento;
   - simbólico/pessoal;
   - operacional de campo.

2. Qual é o domínio principal?
   - NÚCLEO;
   - ASSET;
   - BTC;
   - SIMB;
   - LOTOFA;
   - IRATA/Inspeção;
   - Jurídico;
   - Linguagem;
   - Trabalho/carreira;
   - Meta-skill.

3. Qual é o risco?
   - baixo;
   - médio;
   - alto;
   - crítico.

4. Qual é o estado provável do usuário?
   - normal;
   - cansado;
   - ansioso;
   - acelerado;
   - prático;
   - exploratório;
   - simbólico;
   - operacional;
   - irritado com excesso de complexidade.

5. Qual resposta é adequada?
   - curta;
   - estruturada;
   - documento completo;
   - checklist;
   - decisão por bandas;
   - análise profunda;
   - amortecimento;
   - prompt copiável;
   - arquivo/skill.

## Roteamento

- Baixo risco + tarefa simples: responder direto.
- Técnico/normativo/jurídico: SOURCE-FIRST.
- Experiência pessoal/simbólica: USER-ALIGNED com CORE-LITE.
- Investimento: ASSET + Regime + E-Histórico.
- BTC: BTC-CORE + CRE.
- Lotofácil: LOTOFA + firewall lúdico.
- Decisão crítica: FW-CORE.
- Conflito: CONFLICT-SCAN.
- Sobrecarga: operadores pré-linguísticos.
- Migração/criação de skills: CORE-MIGRATION + HARVEST.
