# Proxies Constitucionais

As pastas com sufixo `_proxy` são versões resumidas e constitucionais
dos cores especializados — não versões operacionais completas.

Função: permitir que o FW-CORE coordene e cheque coerência com os
cores secundários sem carregar toda a operação deles.

Para análise profunda, use o core especializado diretamente:
- simb_core_proxy → .claude/skills/simb-core/
- asset_core_proxy → .claude/skills/asset-core/
- btc_core_proxy → .claude/skills/asset-core/sub/btc_core_cre.md
- irata_proxy → .claude/skills/irata-n3/
- juridico_proxy → .claude/skills/juridico/
- lotofa_proxy → .claude/skills/lotofa/
