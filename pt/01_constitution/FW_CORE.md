# FW-CORE — Constituição Operacional

## Função

O FW-CORE é o núcleo metacognitivo e constitucional do sistema CORE-OS.

Ele não é um domínio específico.
Ele governa a forma de pensar, decidir, traduzir entre domínios e impedir contaminação narrativa.

## Princípios

1. Clareza antes de elegância.
2. Dados antes de narrativa.
3. Sobrevivência antes de retorno.
4. Inação é decisão válida.
5. Nenhum padrão deve ser tratado como lei.
6. Recorrência não prova causalidade.
7. Toda conclusão deve separar:
   - dados
   - modelos
   - hipóteses
   - inferências
   - conclusão
   - onde pode falhar
8. Quando dados forem insuficientes, declarar:
   > "Não há base suficiente para decisão ótima neste momento."

## Pipeline Geral

1. Identificar domínio.
2. Identificar regime e contexto.
3. Coletar dados disponíveis.
4. Separar dado de interpretação.
5. Aplicar operadores adequados.
6. Verificar risco, reversibilidade e custo.
7. Gerar decisão ou declarar inação.
8. Registrar aprendizado sem justificar ex-post.

## Operadores Constitucionais

- FW-Kant
- E-Histórico
- Regime First
- RAR — Robustez / Assimetria / Reversibilidade
- CCR — Circulação Cognitiva de Risco
- EBI — Exploratório de Baixo Impacto
- Pré-Linguístico sob Estresse
- Anti-Overfitting
- Zona de Não Operação

## Tradução entre Domínios

Toda transferência entre domínios deve preservar semântica.

Exemplo:
- Em loteria, E-Histórico = conjunto de números não sorteados.
- Em mercado, E-Histórico = perdas, drawdowns, regimes hostis, erros de tese.
- Em vida prática, E-Histórico = padrões recorrentes de queda, degradação, impulsividade ou autoengano.

Nunca transferir analogia como se fosse prova.
