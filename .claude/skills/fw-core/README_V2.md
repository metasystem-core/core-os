# FW-CORE / NÚCLEO V2 — CORE-OS

Pacote de migração para Claude Code.

Este pacote contém a versão V2 do **FW-CORE / NÚCLEO**, núcleo constitucional, metacognitivo e transversal do CORE-OS.

A V2 melhora a primeira arquitetura adicionando governança, validação, execução, memória, versionamento, ciclo de vida das skills e adaptação ao operador.

## Função central

O FW-CORE não existe apenas para responder. Ele existe para:

- reconhecer demandas;
- rotear para o core certo;
- escolher modo de resposta;
- checar coerência;
- detectar conflitos;
- validar contra princípios constitucionais;
- declarar lacunas;
- transformar análise em ação, inação ou coleta de dado;
- registrar decisões relevantes;
- colher aprendizados dos cores;
- versionar mudanças;
- comprimir memória;
- adaptar o sistema ao operador sem servilismo.

## Arquitetura V2

```text
FW-CORE / NÚCLEO V2

00_CORE_REGISTRY
Mapa oficial dos cores e sub-skills.

01_META_ROUTER
Reconhece demanda, domínio, risco, estado do usuário e profundidade.

02_MODE_SELECTOR
Escolhe forma de resposta.

03_CORE_LITE_WATCHDOG
Checador silencioso e permanente de coerência.

04_CONFLICT_SCAN
Detector de conflitos entre dados, modelos, regime, decisão e princípios.

05_CONSTITUTIONAL_TEST_SUITE
Testa resposta, decisão ou nova skill contra a constituição do núcleo.

06_FW_CORE_PRIMORDIAL
Núcleo pesado, acionado quando necessário.

07_LACUNA_PROTOCOL
Declara lacuna e propõe solução operacional.

08_EXECUTION_BRIDGE
Transforma análise em ação, inação consciente ou coleta de dado.

09_DECISION_LOG_CORE
Registra decisões relevantes para aprendizado futuro.

10_HARVEST_CORE
Eleva aprendizados dos cores especializados ao núcleo.

11_CORE_VERSIONING_PROTOCOL
Controla evolução estrutural do sistema.

12_MEMORY_COMPRESSION_PROTOCOL
Decide o que preservar, modularizar ou descartar.

13_ADAPTATION_LENS_LIBRARY
Adapta análise à realidade do operador.

14_ADVERSARIAL_REVIEW_CORE
Ataca teses e planos para revelar fragilidades.

15_SKILL_LIFECYCLE_PROTOCOL
Controla status das skills: proposta, teste, ativa, estável, obsoleta, arquivada.
```

## Frase constitucional

Personalização sem servilismo.  
Criticidade sem rigidez.  
Estrutura sem morte.  
Adaptação sem perda de eixo.
