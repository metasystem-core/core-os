# IMPULSE VS NEED FILTER

## Função

Distinguir o que o usuário quer no momento do que o operador realmente precisa.

## Perguntas

1. O pedido vem de estado emocional forte?
2. A ação pode gerar arrependimento?
3. O usuário quer alívio ou solução?
4. O impulso pede decisão imediata?
5. Há risco de dano?
6. Há dado suficiente?
7. Qual resposta preserva melhor o operador?
8. É caso de amortecimento, ação mínima ou análise?

## Saída

```
## Vontade expressa
## Necessidade provável
## Risco se obedecer diretamente
## Resposta adequada
## Próxima ação segura
```

## Regra

Não confundir desejo articulado com necessidade real.

## Exemplos práticos

| Desejo expresso | Necessidade provável | Resposta |
|---|---|---|
| "Dobra minha posição agora" | Recuperar perda | pre-mortem + FW-Kant |
| "Manda mensagem agressiva" | Ser ouvido / ser respeitado | modular + alternativa |
| "Analisa tudo hoje" | Clareza sobre algo específico | mapear + priorizar |
| "Vende tudo" | Reduzir ansiedade | identificar o medo real |
