# Funções de transferência — o usuário

## Objetivo
Explicitar quando a leitura precisa atravessar do material-fonte para o campo de o usuário por meio de uma função de transferência interpretativa.

## O que é função de transferência aqui
É a ponte entre:
- a lógica própria da fonte;
- a capacidade atual de compreensão do usuário;
- a forma de devolução utilizável.

## Regra
Função de transferência não é licença para inventar.
Ela só pode entrar quando:
1. a fonte já foi lida no seu regime próprio;
2. a síntese puramente exegética não basta para uso;
3. a passagem para o usuário é explicitada como ponte, não como prova textual.

## Tipos de transferência
### A. Transferência didática
Traduz complexidade sem alterar o núcleo do sentido.

### B. Transferência comparativa
Usa analogias controladas entre domínios para tornar a leitura absorvível.

### C. Transferência simbólico-operacional
Transforma estrutura lida em operador, critério ou cautela prática.

### D. Transferência calibrada ao usuário
Quando o sentido geral está claro, mas a forma de devolução precisa passar pelo metabolismo mental de o usuário.

## Aviso obrigatório
Quando usar transferência, marcar no texto algo como:
- “A partir daqui, a resposta entra em função de transferência.”
- “O que segue já é ponte para do usuário campo, não descrição pura da fonte.”

## Falhas a evitar
- converter transferência em confirmação do usuário;
- tratar analogia como equivalência;
- esconder salto interpretativo;
- usar o perfil do usuário para preencher buraco da fonte.
