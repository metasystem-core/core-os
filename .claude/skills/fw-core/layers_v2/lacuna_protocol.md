# SKILL — Lacuna Protocol

## Função

Declarar falta de base sem travar o sistema, propondo solução operacional.

## Quando usar

- dados insuficientes;
- fonte ausente;
- documento incompleto;
- regime indefinido;
- sinais contraditórios;
- risco alto;
- baixa confiança;
- decisão sem base.

## Tipos de lacuna

1. Dado ausente.
2. Fonte ausente.
3. Contexto ausente.
4. Regime indefinido.
5. Conflito entre sinais.
6. Risco alto demais.
7. Escopo mal definido.
8. Tempo inadequado.
9. Competência de core inadequada.

## Processo

1. Nomear a lacuna.
2. Classificar tipo.
3. Declarar o que não pode ser concluído.
4. Dizer o que ainda pode ser feito.
5. Propor ação: buscar dado, reduzir escala, esperar, tratar como EBI, acionar core ou declarar inação.
6. Definir decisão mínima possível.
