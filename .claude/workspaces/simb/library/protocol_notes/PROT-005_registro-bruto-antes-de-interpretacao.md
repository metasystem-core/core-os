## Definition / Definição

Em paráfrase de Hall: o relato bruto de um sonho — o que foi sonhado, sem reorganização, sem interpretação embutida, sem preenchimento de lacunas — é o dado primário de qualquer trabalho onírico. Qualquer reorganização antes do registro compromete o material. A memória do sonho é frágil e sofre influência imediata do ego desperto que tende a tornar o sonho "mais coerente" do que foi. Registrar primeiro, interpretar depois.

Três tipos de conteúdo que devem ser mantidos separados no registro:
- **Lembrança:** o que o operador efetivamente lembra do sonho.
- **Suposição:** o que parece provável que aconteceu, mas não está claro na memória.
- **Interpolação:** o que o ego desperto inseriu para fazer sentido — deve ser marcado como tal.

## Operational use / Uso operacional

- Aplicar na Fase 1 (DREAM_ANALYSIS_PROTOCOL §5): antes de qualquer pergunta interpretativa, registrar o relato bruto exatamente como o operador apresentou.
- Quando o operador apresenta relato já interpretado ("tive um sonho sobre ansiedade"): perguntar pelas imagens antes de trabalhar com o rótulo.
- Não reorganizar o relato para torná-lo "mais claro" — lacunas, fragmentos e saltos são dados.
- Registrar lacunas explicitamente como `[lacuna]` em vez de preencher com inferência.
- Antes de começar qualquer leitura, perguntar ao operador: "Você quer apenas registrar agora, ou quer uma leitura provisória?"
- Quando a carga afetiva parecer alta: priorizar registro; não oferecer interpretação não solicitada (Fase 15, PATCH-002).

---

## Do not use for / Não usar para

- Não usar como justificativa para não interpretar quando o operador pediu leitura — registro não é recusa.
- Não usar para questionar a memória do operador — registrar o que foi dito, sem avaliar fidelidade.
- Não exigir relato "completo" — fragmentos são dados válidos.
- Não reorganizar o relato para facilitar leitura arquetípica — a ordem do relato é parte do dado.

## Firewalls

- **F-01** (DREAM_ANALYSIS_PROTOCOL §18): não iniciar interpretação sem registro mínimo.
- Lacunas não devem ser preenchidas — `[lacuna]` é dado, não falha.
- Suposição e interpolação devem ser marcadas como tal — não tratadas como lembrança.
- Não reorganizar cronologia do relato para torná-lo linear se o sonho não foi assim lembrado.
