# Princípios do CORE-INTAKE

## Frase constitucional

Todo material entra pelo cesto, mas nada mora no cesto.
O cesto registra, classifica e encaminha.
O core especializado processa.
A library guarda.
O registry prova.

## Princípio de leveza

O cesto deve ser content-light e routing-smart.

## Princípio de economia

Começar por metadados/hash/manifest.
Processar profundamente apenas quando necessário.

## Princípio de especialização proporcional

Todo core relevante pode ter adapter.
Nem todo core precisa de motor pesado.

## Princípio de verdade operacional

O estado real vem do filesystem + registries.
SKILL.md não pode declarar capacidade que os arquivos não sustentam.

## Princípio de não contaminação

Generalizar o cesto não reduz rigor do módulo normativo.
Normas continuam exigindo rastreabilidade, versão, citação, conflito e diff/merge.
