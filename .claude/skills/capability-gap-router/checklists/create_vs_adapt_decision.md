# Criar skill nova vs adaptar skill existente

## Criar skill nova se:

- [ ] domínio é novo;
- [ ] função é recorrente;
- [ ] risco é relevante;
- [ ] não existe skill parecida;
- [ ] há output próprio;
- [ ] há gatilhos claros;
- [ ] há necessidade de watchdog próprio.

## Adaptar skill existente se:

- [ ] domínio já existe;
- [ ] há skill semelhante;
- [ ] falta template;
- [ ] falta subskill;
- [ ] falta output contract;
- [ ] falta watchdog;
- [ ] a lacuna é extensão natural.

## Criar template se:

- [ ] a lógica já existe;
- [ ] saída é repetitiva;
- [ ] não precisa nova capacidade cognitiva.

## Criar código se:

- [ ] tarefa é mecânica;
- [ ] repetitiva;
- [ ] testável;
- [ ] offline;
- [ ] baseada em regras ou estrutura.

## Não criar nada se:

- [ ] caso único;
- [ ] ruído;
- [ ] baixa recorrência;
- [ ] pergunta simples;
- [ ] impulso momentâneo.
