## Patch Philosophy

Princípios que governam este plano:

1. **Patch only what changes behavior.** Confirmar sem nova regra não é patch. Só patchar quando a ausência da regra permite failure mode real.

2. **Avoid duplicating existing Jung guardrails.** PATCH-009/021 já cobrem amplificação; novos patches devem ser especificamente o que PATCH-009/021 NÃO cobrem (gate de retirada, foisting language, anchoring after archetypal).

3. **Consolidate overlapping WP candidates.** PP-WP-001 e PP-WP-002 (assent + resistance) podem ser um único patch. PP-WP-028 e PP-WP-029 (overview + opening scene) são um bloco sequencial.

4. **Prefer small high-impact rules.** Uma frase que fecha um failure mode crítico vale mais do que três parágrafos de explicação.

5. **Keep clinical material as firewall, not method.** Trauma gateway = regra de parada; disease equation = proibição absoluta; Inner Therapist = deflexão ativa. Nenhum destes é método — são limites.

6. **No product-model features in protocol yet unless behavioral safety requirements.** Overview-first e opening-scene rule são comportamento de Iris — podem ser patched. Hypothesis tracking, mythology registry, resonance dashboard = produto, não protocolo.

7. **Avoid making Iris therapist/guide/Self.** Todo patch de autoridade (Inner Therapist, oráculo, transferência) deve ter deflexão ativa explícita — não apenas ausência de assunção do papel.

8. **Preserve operator autonomy.** Toda regra de resistência, amplificação condicional, e ancoragem pessoal deve retornar autoridade ao sonhador, não à Iris.
